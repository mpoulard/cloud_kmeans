Preliminaries: the kk-means algorithm

kk-means clustering aims to partition nn dddimensional observations into kk clusters in which each observation belongs to the cluster with the nearest mean, serving as a prototype of the cluster. This results in a partitioning of the data space into Voronoi cells. The problem is computationally difficult (NP-hard); however, there are efficient heuristic algorithms that are commonly employed and converge quickly to a local optimum. In this Notebook we'll focus on one of them in particular: the Lloyd algorithm.

The kk-means problem can be formalized as follows. Given a set of observations (x1,x2,,xn)(x1,x2,,xn), where each observation is a dd-dimensional real vector, kk-means clustering aims to partition the nn observations into knkn sets S={S1,S2,,Sk}S={S1,S2,,Sk} so as to minimize the within-cluster sum of squares (WCSS) (i.e. variance). The objective is to find:

argminSi=1kxSi||xi||2argminSi=1kxSi||xi||2

The most common algorithm uses an iterative refinement technique. Given an initial set of kk centroids (1)1,,(1)k1(1),,k(1) , the algorithm proceeds by alternating between two steps: in the assignment step, observations are associated to the closest centroid, in terms of squared Euclidean distance; in the update step new centroids are computed based on the new points associated to each centroid. Note: (t)ii(t) stands for the ii-th centroid as of the tt-th iteration. So (1)11(1) is the centroid 1 at iteration 1.

def Matrix_of_distances(X,centroids):
    n = len(X)
    k = len(centroids)
    distance_matrix = np.zeros((n,k))
    for i in range(n):
        for j in range(k):
            distance_matrix[i][j] = np.sqrt( (X[i][0] - centroids[j][0])**2 + (X[i][1] - centroids[j][1])**2 )
    return distance_matrix
    
def Assignment_of_clusters(centroids, distance_matrix):
    
    n = len(distance_matrix)
#     cluster_assignment = np.zeros(n)
    cluster_assignment = np.array([0]*n)
    
    for i in range(n):
        cluster_assignment[i] = int(np.argmin(distance_matrix[i]))
        
    return cluster_assignment

def Computing_the_means(X, cluster_assignment,centroids):
    
    n = len(X)
    k = len(centroids)

    new_centroids = np.array([np.array([0.0,0.0])]*k)
    counter = [0]*k
    
    for j in range(k):
        for i in range(n):
            if (cluster_assignment[i] == j):
                new_centroids[j]+= X[i]
                counter[j] += 1

       
    
    for j in range(k):
        new_centroids[j] = new_centroids[j]/counter[j]
    
    return new_centroids, counter

A simplified analysis of algorithm convergence

One well-known weakness of kk-means is that the algorithm's performance is closely tied with the randomly generated initial centroids' quality. If the algorithm starts with a set of bad inital centers, it will get stuck in a local minimum.

Instead of taking a formal approach to study the convergence of kk-means, let's study it with an experimental approach. One thing we can do is to build a measure of clustering quality: intuitively, a good clustering result should produce clusters in which data points should be very close to their centroids, and very far from other centroids. In this Notebook, we'll look at a metric called the total within Sum of Squares, which is sometimes referred ot as heterogeneity. Mathematically, we define heterogeneity as:

=j=1ki:zi=j||xij||22H=j=1ki:zi=j||xij||22

Where kk denotes the total number of clusters, xixi is the ii-th data point, jj is the jj-th centroid, and ||||22||||22 denotes the squared L2 norm (Euclidean distance) between the two vectors.

A technique for a smart centroid initialization: kk-means++

One effective way to produce good initial centroids to feed to kk-means is to proceed as follows: instead of randomly generating initial centroids, we will try to spread them out in our dd-dimensional space, such that they are not "too close" to eachother. If you are interested in the details of this technique, you should refer to the link to the original research paper below: in summary, the kk-means++ technique allows to improve the quality of the local optima in which kk-means settles, and to reduce the average runtime of the algorithm.

k-means++: the advantages of careful seeding, by David Arthur and Sergei Vassilvitskii (Stanford University) https://dl.acm.org/citation.cfm?id=1283494

A simplified workflow of the kk-means++ approach is as follows:

  • Choose a data point at random from the dataset, this serves as the first centroid
  • Compute the squared euclidean distance of all other data points to the randomly chosen first centroid
  • To generate the next centroid, each data point is chosen with the probability (weight) of its squared distance to the chosen center in the current round, divided by the the total squared distance (this is just a normalization to make sure the probability adds up to 1). In other words, a new centroid should be as far as possible from the other centroids
  • Next, recompute the probability (weight) of each data point as the minimum of the distance between it and all the centers that are already generated (e.g. for the second iteration, compare the data point's distance between the first and second center and choose the smaller one)
  • Repeat step 3 and 4 until we have kk initial centroids to feed to the kk-means algorithm

def distance_to_centroids(centroids,point):
    k = len(centroids)
    distance = min([np.sqrt( (point[0] - centroids[i][0])**2 + (point[1] - centroids[i][1])**2 ) for i in range(k)])
    
    return distance
  

def choose_new_centroid(X,centroids):
    
    n = len(X)
    total_distance = 0
    centroids = list(centroids)
    
    for i in range(n):
        total_distance += distance_to_centroids(centroids,X[i])
        
    probabilities = np.array([distance_to_centroids(centroids,X[i])/total_distance for i in range(n)])
    index_new_centroid = np.argmax(probabilities)
    new_centroid = X[index_new_centroid]
    
    centroids.append(new_centroid)
    
    centroids =np.array(centroids)


    return centroids

Determining the value of k: a simple and visual approach, called the Elbow method

Another problem of kk-means is that we have to specify the number of clusters kk before running the algorithm, which we often don't know a priori. There are many different heuristics for choosing a suitable value for kk, the simplest one being the Elbow method. Essentially, the idea is to run the kk-means algorithm using different values of kk and plot the corresponding heterogeneity. This measure will decrease as the number of clusters increases, because each cluster will be smaller and tighter. By visual inspection of the plot heterogeneity vs. kk, we will (hopefully!) see that the curve flattens out at some value of kk: this is what we call an "elbow", and we'll select the value of kk corresponding to the "elbow" position.

Distributed kk-means with PySpark

We're now ready to work on a distributed implmentation of the kk-means algorithm, using the PySpark API.

By now, you should be rather familiar with the kk-means algorithm, which means we can focus on its parallel and distributed design.

Distributed algorithm pseudo code

The basic idea of distributed kk-means is as follows: data points to be clustered should be stored as a distributed dataset, namely a RDD. As in the Notebook on distributed SGD, we will take a shortcut and avoid using HDFS RDDs: rather, we'll use sklearn to generate the data points, similary to the serial version of the algorithms, then use the parallelize() method to create an RDD, and determine the number of partitions.

We also need to manipulate the centroids array: indeed, all machines should hold a copy of the centroid vector, such that they can proceed independently and in parallel in the first phase of the kk-means algorithm, that is the assignment step. Specifically, every worker has a set of data points, and it will use a copy of the centroid vector to compute cluster assignement: we compute the distance between each data point and each centroid, to assign data points to their closest centroid.

Once the assignement step is done, we need to recompute new centroids based on the assignement, that is, we execute the update step. Clearly, we will need to shuffle data over the network such that we will have, for each current centroid, the list of all data points that have been assigned to it. If you think about it, this problem should be familiar!! This is very similar to what we do in the Word Count example. As such, you will need to make sure the output of the update step is cast to a <key, value> type, where the key corresponds to a centroid identifier, and the value contains the list of data points associated to that centroid. The framework will take care of the distributed group by operation, and organize data according to the semantic of our algorithm.

NOTE: since we will (potentially) work on large dataset sizes, we don't want our algorithm to return the final assignement after convergence, for otherwise we would need to collect a large amount of data in the driver machine, which has a finite and somehow limited amount of RAM.

The pseudo code of the algorithm you need to implement is as follows:

datapoints = # Use sklearn, as usual, and work on blobs
